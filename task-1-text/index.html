Развитие веб-технологий: От статичных страниц к Web3

Краткий экскурс в историю фронтенда, разбор ключевых элементов современной верстки и методологий разработки.

Содержание статьи

    1. Эволюция стандартов

    В начале пути веб состоял из простых текстовых документов. Сегодня HTML предоставляет огромный выбор семантических тегов, которые помогают поисковым роботам и скринридерам правильно понимать структуру страницы.

    Семантика — это не просто правила хорошего тона, это фундамент доступности веб-интерфейсов для всех категорий пользователей.

    — Неизвестный веб-технолог

    2. Инструменты и методологии

    Для поддержки порядка в больших проектах разработчики используют строгие правила. Самой популярной и надежной остается методология БЭМ (Блок, Элемент, Модификатор). Она изолирует стили и делает код предсказуемым.

    Сравнение подходов к стилизации

    Критерий Чистый CSS (Каскад) Методология БЭМ
    Скорость старта Высокая Средняя
    Изоляция стилей Отсутствует Полная
    Масштабируемость Низкая Высокая

    3. Демонстрация редких элементов

    Хороший верстальщик знает и умеет использовать специфические теги для списков определений и форматирования кода:

    Блок
    Независимый интерфейсный компонент, который может быть переиспользован.
    Элемент
    Составная часть блока, которая не может использоваться в отрыве от него.

    Пример inline-кода и блока кода

    Чтобы сделать текст жирным, мы используем тег <strong>, а не стили font-weight: bold напрямую в разметке. 

    .article-page__title {
  font-size: 2rem;
  margin-bottom: 1.5rem;
}
